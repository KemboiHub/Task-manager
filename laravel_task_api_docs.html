GET /api/tasks List all tasks ▾
Try it
Rules
→Sorted by priority: high → medium → low (using FIELD() in MySQL)
→Then sorted by due_date ascending within each priority group
→Optional ?status=pending|in_progress|done query parameter
→Returns meaningful JSON message when no tasks exist

→priority must be one of: low, medium, high
→status is automatically set to pending on creation
PATCH /api/tasks/{id}/status Progress task status ▾
Try it
Flow
STATUS FLOW
pending → in_progress → done
→Status can ONLY progress forward: pending → in_progress → done
→Cannot skip a step (e.g. pending → done is rejected)
→Cannot revert (e.g. done → pending is rejected)
→Once done, no further status changes are possible
→Returns 422 with a clear message on invalid transitions
DELETE /api/tasks/{id} Delete a done task ▾
Try it
Rules
→Only tasks with status done can be deleted
→Attempting to delete a pending or in_progress task returns 403 Forbidden
→Non-existent task returns 404 Not Found
→Returns 200 with success message on deletion
GET /api/tasks/report Daily task report ▾
Try it
Rules
→Requires ?date=YYYY-MM-DD query parameter
→Returns counts grouped by priority (high/medium/low) × status (pending/in_progress/done)
→Filters tasks by due_date matching the given date
→Always returns all 9 cells even if count is 0
